監視ツール
- Datadog
  - メトリクスのデータはDatadogが管理してくれる
    - メトリクスを管理するリソースや運用コストは少なくなる
    - ホストやコンテナごとに
- Prometheus

それぞれラベルとタグを使い自由な粒度で監視する対象をクエリすることができ、コンテナのスケーラビリティに伴うモニタリングの複雑性、運用負荷を抑制可能

Datadogのアーキテクチャ
- DatadogをKubernetes上にデプロイする際は、DaemonSetを利用してDatadog Agentを各ノード上で起動する
- デプロイされたAgent以下のメトリクスを取得する
  - 各ホストのCPU使用率やディスク使用率といったメトリクス
  - 各ノードのコンテナ使用率のメトリクス
- DatadogをKubeernetes上で連携して起動するにはコンテナにいくつかの環境変数を渡す必要がある
- クラスタレベルのメトリクスを収集するコンポーネント（kube-stat-metrics）を追加でインストールすることが推奨されている
  - → Deploymentで起動しているPod数や要求起動数などのメトリクスを扱うことができるように

datadog.tags
- 複数のKubernetesクラスタや仮想マシンなどでDatadogを利用している場合に、メトリクスの分類をするために使うタグを追加で付与しておく
